intersectValue

intersectValue (callable $valComparator, callable $keyComparator, iterable ...$arrays): ScalarArray|ObjectArray

Compute intersection of values, considering keys or not.

see array_intersect  array_intersect_assoc  array_intersect_uassoc  array_uintersect  array_uintersect_assoc  array_uintersect_uassoc.

intersect() returns an array containing all the values of $scalarObject->val that are present in all the arguments. Note that keys are preserved.

Parameters

$array2
An array to compare values against.
...
A variable list of arrays to compare.

Return Values

Array containing all of the values in $scalarObject->val whose values exist in all of the parameters. 

            $array1 = ["a" => "green", "red", "blue"];
            $array2 = ["b" => "green", "yellow", "red"];
            intersect($array1, $array2); // output: ["a" => "green", 0 => "red"]
Note: Two elements are considered equal if and only if (string) $elem1 === (string) $elem2. In words: when the string representation is the same.

assocIntersect

assocIntersect() returns an array containing all the values of $scalarObject->val that are present in all the arguments. Note that the keys are used in the comparison unlike in intersect().

Parameters

$array2
An array to compare values against.
...
A variable list of arrays to compare.

Return Values

An associative array containing all the values in $scalarObject->val that are present in all of the arguments. 

            $array1 = ["a" => "green", "b" => "brown", "c" => "blue", "red"];
            $array2 = ["a" => "green", "b" => "yellow", "blue", "red"];
            assocIntersect($array1, $array2); // output: ["a" => "green"]

In our example you see that only the pair "a" => "green" is present in both arrays and thus is returned. The value "red" is not returned because in $scalarObject->val its key is 0 while the key of "red" in $array2 is 1, and the key "b" is not returned because its values are different in each array.

The two values from the key => value pairs are considered equal only if (string) $elem1 === (string) $elem2 . In other words a strict type check is executed so the string representation must be the same.

userAssocIntersect

userAssocIntersect() returns an array containing all the values of $scalarObject->val that are present in all the arguments. Note that the keys are used in the comparison unlike in intersect().

Parameters

$array2
First array to compare keys against.
...
Variable list of array arguments to compare values against.
$keyComparator
The comparison function must return an integer less than, equal to, or greater than zero if the first argument is considered to be respectively less than, equal to, or greater than the second.

Return Values

Returns the values of $scalarObject->val whose values exist in all of the arguments.

            $array1 = ["a" => "green", "b" => "brown", "c" => "blue", "red"];
            $array2 = ["a" => "GREEN", "B" => "brown", "yellow", "red"];
            userAssocIntersect($array1, $array2, "strcasecmp"); // output: ["b" => "brown"]
